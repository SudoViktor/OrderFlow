OrderFlow Terminal System
📝 Опис проекту
OrderFlow — це професійна термінальна система для автоматизації бізнес-процесів (CRM/ERP), розроблена для оптимізації обліку клієнтів, товарних запасів та управління замовленнями. Проект фокусується на швидкості, чистоті даних та надійній архітектурі, що дозволяє бізнесу масштабувати операційну діяльність.

🎯 Задача
Створити високоефективну систему для адміністрування бізнес-процесів, яка забезпечує:

Безпечну авторизацію користувачів.

Зручне управління клієнтською базою.

Автоматизоване створення складних замовлень (з розрахунком суми).

Високу швидкість роботи завдяки пагінації даних на стороні сервера.

🛠 Технологічний стек
Backend:
Flask (Python): Легкий та потужний мікрофреймворк для побудови RESTful API.

SQLAlchemy 2.0: Об'єктно-реляційне відображення для безпечної та ефективної роботи з базою даних.

Flask-JWT-Extended: Забезпечення безпеки через JWT-токени (JSON Web Tokens).

Flask-CORS: Гнучке налаштування крос-доменних запитів для стабільної комунікації з фронтендом.

Frontend:
Vanilla JavaScript (ES6+): Використання модульної структури для підтримки чистого та зрозумілого коду без надлишкових бібліотек.

CSS3 (Flexbox, Grid): Створення адаптивного, "чистого" інтерфейсу в стилі техно-мінімалізму.

Fetch API: Асинхронна взаємодія з сервером.

🚀 Ключові можливості
Модульна архітектура: Система розділена на логічні блоки (Замовлення, Клієнти, Товари), що дозволяє легко додавати новий функціонал.

Оптимізація запитів: Впроваджено серверно-орієнтовану пагінацію, що мінімізує навантаження на мережу та базу даних.

Розумний кошик: Автоматичний розрахунок total_price на етапі створення замовлення та перевірка цілісності даних.

Безпека: Реалізовано перехоплення неавторизованих запитів (401 Unauthorized) з автоматичним завершенням сесії.

🏗 Архітектура даних
Система побудована на зв'язках:

Order (Замовлення) <-> Client (Клієнт): One-to-Many.

Order (Замовлення) <-> OrderItem (Позиції): Many-to-Many через проміжну таблицю з підтримкою cascade="all, delete-orphan" для забезпечення чистоти даних.

🛠 Інструкція з розгортання

Встановлення залежностей: pip install -r requirements.txt

Конфігурація: Налаштуйте JWT_SECRET_KEY у файлі конфігурації.

Запуск: python run.py
